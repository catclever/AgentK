# Agent K: Architecture V2 Blueprint (The Decoupled Paradigm)

这是一份彻底重构 Agent K 宇宙的核心架构蓝图。我们完全抛弃了笨重的 WebContainer 沙盒，转而采用“纯前端注入 + 独立模块化交互 + 本底连接”的极简架构。

---

## 一、运行模式的分治 (Runtime Modes)

系统将严格区分“开发态”与“生产态”，但两者的底层代码完全一致：

1. **工作台模式 (Inside `shoggoth`)**
   * **不再需要 WebContainer**。当应用在 `shoggoth` 内加载时，底层的构建和数据直接**直连本地运行的真实后端服务**（如本地跑着的 Vite Server 和 Descartes API）。
   * 享受本地真实的磁盘 I/O 速度，0秒瞬间启动，彻底消灭“沙盒预热税”。

2. **独立运行模式 (Direct App Runtime)**
   * 当用户直接打开打包后的 Demo App 时，这是一个**纯血的浏览器单页应用 (SPA)**。
   * 数据持久化退回浏览器的 IndexedDB/RxDB。没有任何 Node.js 环境参与。

---

## 二、四大基础模块的重新定义 (The Four Pillars)

我们废除集中的 UI 状态管理，走向**微前端/独立挂载**的思想。每个核心库负责自己的功能，**并自己维护自己的悬浮交互面板**，`shoggoth` 不再负责庞大而脆弱的位置布局维护，只提供底层容器。

### 1. 探针与大脑：`eye` (The Smart Probe)
* **核心职责**：它是永远伏击在浏览器环境（`window`）里的刺客。负责强劫持原生浏览器的所有报错、抓取 DOM 结构与组件状态。
* **界面表现**：它自带一个**AI 聊天/分析专属的独立交互面板**（比如一个悬浮的机器人对话框）。**它绝对不包含黑漆漆的命令行终端界面 (No Terminal)**。
* **适用性**：无论应用是独立运行在生产环境，还是在 `shoggoth` 里被调试，`eye` 都可以被无缝注入。遇到错误时它内部消化，并展示在其自有的 AI 交互面板内。

### 2. 终端与控制台：`gladius` (The Execution Console)
* **核心职责**：以前它负责管沙盒环境，现在它的定位涅槃为**“终端与控制台界面”**。
* **脱离 `shoggoth` 独立使用时**：它就像一个内置在网页里的“更强大的 Chrome 开发者控制台 (F12)”。它可以展示具体的业务日志、网络报文，甚至允许你在里面输入指令直接向页面注入执行逻辑。
* **在 `shoggoth` 里使用时**：它化身为连接本地真实后端的“标准输出展示板”。由于取消了 WebContainer，它现在用来 `pipe` 接收来自本地 Node 真实后端的构建流日志（以及双向操作通道）。

### 3. 数据通信的独立性 (Independent Data Tunnels)
既然 `eye` 和 `gladius` 是可以分开独立 `npm install` 与注入的包，它们的通信链路必须是**互相物理隔离的**：
* `gladius` 与后端的连接，专注打通“日志与控制台指令流”。
* `eye` 与后端的连接，专注打通“错误实体、AST JSON 以及大模型结构化指令流”。
* 后端的数据流可以分别独立发给需要它的渠道。两者互不干扰，即使只装一个也能跑。

### 4. 万物容器：`shoggoth` (The Empty Canvas)
* **退位让贤**：`shoggoth` 应当摒弃那些复杂的拖拽布局（DraggableWindow 等状态计算），将弹窗的位置管理全部归还给 `eye` 和 `gladius` 组件自身的内部实现。
* **它只做什么？**：它彻底退化为一个**纯粹的空白画板和集成器**。负责提供占据全屏的 Iframe Preview，并将 `EyeUI` 和 `GladiusConsole` 两个独立组件引入挂载在网页顶层即可。

## 附录：包组织与引用关系 (Package Architecture)
整个 Mono-repo 体系遵循严格的单向依赖与功能解耦原则：

1. **`core` (独立包：引擎层)**
   * **角色**：核心的运行时，专注提供诸如 `Renderer`、实体 Hooks、CQRS 流等黑魔法。
   * **依赖**：不依赖任何兄弟包。它会被作为 npm 依赖**注入到最终用户的产品代码里**。

2. **`libs/` (仓库目录：工具集)**
   * **角色**：存放一组**相互独立**的开发套件组件包（如 `eyes`、`gladius`、`tentacles`，甚至未来的管理工具）。
   * **依赖**：它们是纯粹的 UI / Agent 通信组件，**绝对不需要引用 `core`**。

3. **`shoggoth` (独立包：IDE 宿主)**
   * **角色**：开发工作台 Shell，利用 JSON 自举动态映射加载界面。
   * **依赖**：它需要按需引用 `libs` 里提供的工具组件来拼装界面。
   * **彻底避开 Core**：`shoggoth` 本身**不需要引用 `core`**。即便它用了 JSON 渲染界面的理念，也可以在 `shoggoth` 层利用 TypeScript 的鸭子类型（Duck-Typing）维护一份极简的 `ComponentInstance` 类型映射定义，从而完全隔离针对目标应用的沉重 `core` 运行时包。
